1. PROJECT OVERVIEW
================================================================================

Project Name    : AutoMarketer AI
Type            : Web-Based AI-Powered Marketing Automation Platform
Category        : Final Year Project (FYP)
Current Stage   : Frontend Development Complete (React + TypeScript)
AI Model        : Kimi K2.5 (Moonshot AI) via Hugging Face Inference API
Backend         : n8n Webhook Integration (planned) / Mock Mode (active)

AutoMarketer AI is an intelligent marketing automation platform that leverages
artificial intelligence to help businesses and marketers create, manage, and
optimize their digital marketing campaigns. The platform uses Moonshot AI's
Kimi K2.5 large language model through the Hugging Face Inference API to
provide AI-powered features such as content generation, competitor analysis,
audience profiling, and lead scoring.

The frontend is fully built using React 19, TypeScript, and Vite, with a
modern dark/light theme design system. The AI integration layer is designed
with a dual-mode architecture â€” it can run in MOCK MODE (returning simulated
AI responses for demos) or LIVE MODE (sending real requests to the AI model).

2. PROBLEM STATEMENT
================================================================================

Small and medium businesses face significant challenges in digital marketing:

- Creating engaging content across multiple social media platforms is
  time-consuming and requires specialized expertise.
- Understanding competitors' strategies and identifying market opportunities
  requires expensive tools and manual research.
- Scoring and qualifying leads requires intuition and experience that
  small teams often lack.
- Building detailed audience personas for targeted campaigns is complex
  and resource-intensive.
- Managing campaigns across channels (email, social, search) requires
  multiple disconnected tools.

AutoMarketer AI addresses these challenges by providing an all-in-one
AI-powered platform that automates and optimizes the entire marketing workflow.

7. FEATURE BREAKDOWN (ALL 16 FEATURES)
================================================================================

Features are categorized into three tiers based on their AI integration:

TIER 1: AI-POWERED FEATURES (Core - Use Kimi K2.5)
---------------------------------------------------
These features directly call the AI service and represent the core innovation
of the project.

  1. CONTENT STUDIO (Content Generator)
     - Route: /generate
     - AI Functions Used: generateMarketingCopy(), generateCampaignStrategy(),
       generateSeoKeywords(), optimizeContent(), generateMarketingImage()
     - Description: The main AI workstation. Users input a topic, select
       platform (Twitter/LinkedIn/Instagram/Facebook), choose a tone
       (Professional/Casual/Humorous/Urgent/Inspirational), specify target
       audience, and the AI generates platform-optimized marketing posts.
     - Sub-features:
       a) Post Generation - Creates social media posts
       b) Campaign Strategy - Generates full marketing strategies
       c) SEO Research - Finds keywords, content ideas, competitor URLs
       d) Content Optimization - Improves existing copy
       e) Image Generation - Creates marketing visuals (via Unsplash mock)

  2. AUDIENCE BUILDER
     - Route: /audience
     - AI Functions Used: generateAudiencePersona()
     - Description: Generates detailed buyer personas using AI. Users input
       product name, industry, and region. The AI creates a complete persona
       including demographics, goals, frustrations, motivations, and
       preferred marketing channels.

  3. COMPETITOR ANALYSIS
     - Route: /competitors
     - AI Functions Used: analyzeCompetitor()
     - Description: Performs automated SWOT (Strengths, Weaknesses,
       Opportunities, Threats) analysis on competitors. Users enter a
       competitor name and industry, and the AI returns a comprehensive
       competitive intelligence report with strategic advice.

  4. LEAD MANAGEMENT
     - Route: /leads
     - AI Functions Used: analyzeLeadScore()
     - Description: AI-powered lead scoring system. Each lead has a name,
       email, source, and status. The AI analyzes lead data and assigns
       an engagement score (0-100) with reasoning for the score.

8. AI INTEGRATION DETAILS (Kimi K2.5 VIA HUGGING FACE)
================================================================================

FILE: src/services/aiService.ts (224 lines)

The AI service uses a SINGLETON PATTERN with the ApiClient class. It operates
in two modes controlled by the REACT_APP_USE_MOCKS environment variable:

MODE 1: MOCK MODE (Default - REACT_APP_USE_MOCKS=true)
-------------------------------------------------------
- Returns realistic pre-built mock data after a simulated delay
- Delays vary per function to simulate "thinking" time:
  - generateMarketingCopy()    : 1500ms delay
  - generateCampaignStrategy() : 4000ms delay (simulates deep thinking)
  - analyzeLeadScore()         : 2500ms delay
  - analyzeCompetitor()        : 4000ms delay (simulates deep thinking)
  - optimizeContent()          : 1500ms delay
  - generateAudiencePersona()  : 3500ms delay
  - generateSeoKeywords()      : 3000ms delay
  - generateMarketingImage()   : 3000ms delay

MODE 2: LIVE MODE (REACT_APP_USE_MOCKS=false)
---------------------------------------------
- Sends HTTP POST requests to n8n webhook URL
- n8n workflow routes the request to Hugging Face Inference API
- Uses Kimi K2.5 model (moonshotai/Kimi-K2.5) for text generation
- Falls back to mock data if the API call fails (graceful degradation)
- Request format: JSON with { action: string, ...payload }

API FUNCTIONS:
+----------------------------+------------------+---------------------------+
| Function Name              | Action ID        | Returns                   |
+----------------------------+------------------+---------------------------+
| generateMarketingCopy()    | generate_copy    | string[] (3 posts)        |
| generateCampaignStrategy() | generate_strategy| CampaignStrategy object   |
| analyzeLeadScore()         | analyze_lead     | {score, reason}           |
| analyzeCompetitor()        | analyze_competitor| SwotAnalysis object      |
| optimizeContent()          | optimize_content | OptimizationResult object |
| generateAudiencePersona()  | generate_persona | Persona object            |
| generateSeoKeywords()      | generate_seo     | SeoResult object          |
| generateMarketingImage()   | (direct)         | Image URL string          |
+----------------------------+------------------+---------------------------+

ENVIRONMENT CONFIGURATION (.env):
- REACT_APP_N8N_WEBHOOK_URL : n8n production webhook endpoint
- REACT_APP_USE_MOCKS       : true/false toggle for mock mode
- HUGGINGFACE_API_TOKEN     : API authentication token for Hugging Face
- PORT                      : Development server port (default: 3000)

9. HOW EACH AI FEATURE WORKS (DETAILED FLOW)
================================================================================

--- CONTENT STUDIO (The Main AI Feature) ---

Step 1: User opens Content Studio (/generate)
Step 2: User selects a tab:
        - "Generate" tab: Create new social media posts
        - "Strategy" tab: Generate campaign strategies
        - "SEO" tab: Research keywords and content ideas
        - "Optimize" tab: Improve existing marketing copy
Step 3: User fills in the form:
        For Generate: Topic, Platform, Tone, Target Audience
        For Strategy: Product Name, Campaign Goal
        For SEO: Topic, Niche
        For Optimize: Original Text, Optimization Goal
Step 4: User clicks the generate button
Step 5: The useMarketingStudio() hook calls the appropriate aiService function
Step 6: Loading state shows a "thinking" animation
Step 7: AI response is displayed in the Results Terminal panel
Step 8: User can copy generated content to clipboard or generate images

--- AUDIENCE BUILDER ---

Step 1: User navigates to /audience
Step 2: User enters: Product Name, Industry, Region
Step 3: Clicks "Generate Persona"
Step 4: AI creates a detailed buyer persona with:
        - Name, Age Range, Occupation, Income Level
        - Bio describing the persona
        - Goals (what they want to achieve)
        - Frustrations (pain points)
        - Motivations (what drives them)
        - Preferred Channels (where to reach them)
Step 5: Persona card is displayed with all details

--- COMPETITOR ANALYSIS ---

Step 1: User navigates to /competitors
Step 2: Enters competitor name and industry
Step 3: Clicks "Analyze Competitor"
Step 4: AI performs SWOT analysis:
        - Strengths: What the competitor does well
        - Weaknesses: Where they fall short
        - Opportunities: Market gaps to exploit
        - Threats: Risks to be aware of
        - Strategic Advice: Recommended action plan
Step 5: SWOT matrix is displayed in a 2x2 grid with advice

--- LEAD MANAGEMENT ---

Step 1: User navigates to /leads
Step 2: Sees a table of leads with name, email, source, status
Step 3: Clicks "AI Score" button on any lead
Step 4: AI analyzes the lead's engagement data
Step 5: Returns a score (0-100) and reasoning
Step 6: Score badge updates with color coding:
        - Green (80+): Hot lead
        - Yellow (60-79): Warm lead
        - Red (<60): Cold lead

10. USER AUTHENTICATION SYSTEM
================================================================================

FILE: src/contexts/AuthContext.tsx

The authentication system uses React Context API to manage user state
across the application.

Features:
- Login with email and password (mock authentication)
- Registration with name, email, and password
- Protected routes (redirect to /login if not authenticated)
- Persistent session state during browsing
- Logout functionality with redirect

Components:
- AuthProvider: Wraps the entire app with auth context
- useAuth(): Custom hook to access auth state and actions
- ProtectedLayout: Higher-order component that checks authentication

Flow:
  1. User visits any protected route
  2. ProtectedLayout checks isAuthenticated flag
  3. If not authenticated: redirects to /login
  4. User enters credentials on Login page
  5. On success: isAuthenticated = true, user object is set
  6. User is redirected to Dashboard (/)
  7. Auth state persists across page navigation

13. HOW TO RUN THE PROJECT
================================================================================

PREREQUISITES:
  - Node.js (v18 or later)
  - npm (comes with Node.js)

INSTALLATION:
  1. Open terminal in the project directory
  2. Run: npm install
  3. This installs all dependencies listed in package.json

DEVELOPMENT MODE:
  Run: npm run dev
  - Starts Vite dev server on http://localhost:3000
  - Hot Module Replacement (HMR) enabled
  - Changes reflect instantly in the browser

PRODUCTION BUILD:
  Run: npm run build
  - Creates optimized production bundle in dist/ folder
  - TypeScript compiled, code minified, assets optimized
  - Total bundle size: ~450 KB gzipped

PREVIEW PRODUCTION BUILD:
  Run: npm run preview
  - Serves the production build locally for testing

ENVIRONMENT CONFIGURATION:
  Edit .env file to configure:
  - REACT_APP_USE_MOCKS=true  (for demo/mock mode)
  - REACT_APP_USE_MOCKS=false (for live AI mode)
  - HUGGINGFACE_API_TOKEN=your_token_here
  - REACT_APP_N8N_WEBHOOK_URL=your_webhook_url

DEFAULT LOGIN CREDENTIALS (Mock Auth):
  - Any email and password will work in mock mode
  - Register page creates a new mock user
